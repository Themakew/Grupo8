Main Description

Juntos, os Requisitos Suplementares e os Casos de Uso definem todos os requisitos do sistema. Os Casos de Uso descrevem os requisitos comportamentais do sistema, e os Requisitos Suplementares descrevem todos os outros requisitos que não são capturados nas Especificações de Caso de Uso. Ao fazer esta distinção simplificamos a manutenção.

Os Requisitos Suplementares podem ser categorizados de acordo com o modelo FURPS+ (Funcional, Usabilidade, Confiabilidade, Desempenho, Suportabilidade + Restrições). Para mais informações sobre esta classificação, veja Concept: Requisitos Suplementares.
