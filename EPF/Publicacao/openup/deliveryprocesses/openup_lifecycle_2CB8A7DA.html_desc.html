Delivery Process: Ciclo de Vida do OpenUP
Este processo de entrega define um processo de desenvolvimento de software completo que suporta os princípios fundamentais do OpenUP. Foi projetado para suportar equipes pequenas e co-localizadas, em suas atividades diárias.

Purpose

Este processo de entrega é direcionado para:

  • Promover a colaboração (tanto interna quanto externa à equipe), para alinhar os interesses e compartilhar o entendimento.
  • Ajudar a equipe focar na arquitetura, o mais cedo possível, para reduzir o risco e organizar o desenvolvimento
  • Ajudar a equipe a equilibrar as prioridades concorrentes para maximizar o benefício aos Stakeholders
  • Ajudar a equipe a evoluir o produto para continuamente obter feedback e promover melhorias
  • Permitir que os gestores de projeto monitorem o status com base em metas
  • Permitir que os membros da equipe entendam como realizar o seu trabalho para atingir as metas do projeto

Description

O OpenUP é um processo iterativo com as iterações distribuídas por quatro fases: Concepção, Elaboração, Construção e Transição.

Cada fase pode ter quantas iterações forem necessárias, dependendo do grau de incerteza do domínio de negócio, da tecnologia a ser utilizada, da complexidade arquitetural, do tamanho do projeto, etc.

Para oferecer um início rápido para as equipes planejarem suas iterações, o OpenUP fornece templates de estrutura de decomposição de trabalho (EDT) para cada iteração e um modelo de EDT para um processo de desenvolvimento completo.

As iterações podem ter duração variável, dependendo das características do projeto. Iterações com um mês de duração são normalmente recomendadas, pois este período fornece:

  • Uma quantidade razoável de tempo para que os projetos possam entregar significantes incrementos de funcionalidade.
  • Rápido e freqüente Feedback dos clientes.
  • Oportuna gestão de riscos e problemas no decurso do projeto.

O OpenUP destina-se a oferecer orientação de processo para equipes pequenas e co-localizadas.
