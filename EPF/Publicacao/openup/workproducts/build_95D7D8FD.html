Description
Main Description

Esta versão de trabalho do sistema é o resultado da colocação da implementação do sistema em um processo de construção (normalmente um script automatizado de construção) que cria uma versão executável do sistema, ou que funcione. Esta versão executável do sistema terá tipicamente uma quantidade de arquivos de suporte que também são considerados como parte deste artefato composto.
Key Considerations

Em um ciclo de vida iterativo, cada construção deve evoluir a partir da construção da iteração anterior, adicionando mais funcionalidade e aumentando a qualidade.

A finalidade das primeiras construções que minimizam ou eliminam um risco, ou verificam decisões arquiteturais, é conseguir construções consistentemente estáveis nas iterações posteriores.
Tailoring
Reasons for not needing

Sempre será necessária uma versão operacional do sistema.
Representation Options

Este produto de trabalho é quase sempre um produto composto, feito de numerosas partes necessárias para tornar o sistema executável. Conseqüentemente uma construção é mais do que arquivos executáveis; ela inclui arquivos de configuração, arquivos de ajuda e repositórios de dados que serão agrupados, resultando no produto que será executado pelos usuários. As partes específicas irão variar de acordo com a tecnologia em uso.
