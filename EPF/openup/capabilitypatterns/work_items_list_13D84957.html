Work Product Descriptor (Artifact): Lista de Itens de Trabalho
Este artefato contém uma lista de todo trabalho agendado para ser feito dentro do projeto, bem como o trabalho proposto que pode afetar o produto neste ou em projetos futuros. Cada Item de trabalho pode conter referências a informações que sejam relevantes para realizar o trabalho descrito no item.

Main Description

Este artefato fornece um ponto focal para toda a equipe:

  • Fornece uma lista que contém todas as solicitações de melhorias ou capacidades adicionais para a aplicação. Note que alguns destes pedidos podem nunca ser implementados, ou ser implementados em projetos posteriores.
  • Fornece uma lista de todo trabalho a ser estimado, priorizado e atribuído dentro do projeto. A lista de riscos é priorizada separadamente.
  • Fornece um lugar único para que à equipe de desenvolvimento saiba quais Micro-Incrementos precisam ser entregues, possui referências do material necessário para realizar o trabalho e para relatar o progresso feito.

Estes são os itens de trabalho típicos desta lista.

  • Casos de Uso e referências às especificações de caso de uso
  • Requisitos suplementares
  • Solicitações de mudança e de melhoria
  • Defeitos
  • Tarefas de desenvolvimento

Os itens de trabalho podem ter um escopo muito grande, especialmente os direcionados a melhorias, tais como "Apoio ao Planejamento Financeiro" para uma aplicação financeira pessoal. Para permitir que aplicação seja desenvolvida em micro-incrementos, os Itens de Trabalho são analisados e subdivididos em itens de trabalho menores de forma que possam ser atribuídos a uma iteração, tais como um cenário de caso de uso para "Calcular lucro líquido". Uma subdivisão adicional pode ser necessária para identificar as tarefas apropriadas a serem atribuídas aos desenvolvedores, tais como "Desenvolver Interface de Usuário para Calcular o Lucro Líquido". Isto significa que os itens de trabalho normalmente têm relacionamentos pai-filho, onde o nível mais baixo é uma especificação e um dispositivo de rastreamento para os micro-incrementos.

Os Itens de Trabalho devem conter estimativas, veja Guideline: Gerenciando Itens de Trabalho e Guideline: Estimativa Ágil.
