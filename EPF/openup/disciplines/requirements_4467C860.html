Discipline: Requisitos
Esta disciplina explica como elicitar, analisar, especificar, validar e gerenciar os requisitos para o sistema a ser desenvolvido.

Main Description
d

O propósito desta disciplina é:

  • Entender o problema a ser resolvido
  • Entender as necessidades dos Stakeholders (o que os usuários desejam)
  • Definir os requisitos para a solução (o que o sistema tem que fazer)
  • Definir os limites (escopo) do sistema
  • Identificar interfaces externas ao sistema
  • Identificar restrições técnicas na solução
  • Fornecer a base para o planejamento das iterações
  • Fornecer a base inicial para a estimativa de custo e cronograma

Para conseguir estas metas, é importante compreender a definição e o escopo do problema que você está tentando resolver. Identifique os Stakeholders e defina o problema a ser resolvido.

Concordando com o problema a ser resolvido, os Requisitos para o sistema são elicitados, organizados, analisados, validados e especificados.

Durante todo o ciclo de vida, você gerencia as mudanças nos requisitos.

A disciplina de Requisitos está relacionada às outras disciplinas das seguintes formas:

  • As disciplinas de Arquitetura e Desenvolvimento obtém suas principais entradas a partir da disciplina de Requisitos.
  • A disciplina de Teste valida o sistema de acordo com os requisitos.
  • A disciplina de Gestão de Configuração e Mudança fornece os mecanismos para controlar as mudanças nos requisitos.
  • A disciplina de Gestão de Projeto planeja o projeto e atribui os requisitos a cada iteração analisando os requisitos priorizados e atribuindo o trabalho.
