Checklist: Caso de Teste
Esta lista de verificação contém questões que ajudam a avaliar se os casos de teste foram criados de forma consistente e completa.
Relationships
Related Elements
  • Caso de Teste
  • Teste
Check Items
Geral
  • O caso de teste identifica o requisito a ser validado? Esta identificação pode ser uma convenção informal de nomenclatura ou uma matriz formal de rastreamento de requisitos.
  • O caso de teste referencia as precondições e pós-condições que se aplicam a ele?
Nome
  • O nome do Caso de Teste é exclusivo?
  • O nome expressa uma condição de teste ou um resultado esperado?
  • O nome é inequívoco para os Stakeholders?
Descrição Resumida
  • A condição lógica de teste está claramente identificada na descrição?
  • A descrição expressa claramente o resultado esperado?
  • O resultado esperado está expresso como um resultado concreto?
  • Um leitor casual pode distinguir este caso de teste de um outro similar?
Dados de Teste Necessários
  • O caso de teste aponta os tipos de dados de teste necessários para implementar um script de teste detalhado?
  • Os tipos de dado de teste são únicos e explicados com suficiente qualidade?
