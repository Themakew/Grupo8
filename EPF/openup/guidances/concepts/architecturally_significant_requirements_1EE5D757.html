Concept: Requisitos Arquiteturalmente Significantes
Alguns requisitos têm um impacto profundo na arquitetura da solução e necessitam de atenção especial.

Main Description

Nem todos os requisitos têm igual significância na arquitetura. Alguns desempenham um papel importante na determinação da arquitetura do sistema, mas outros não.

Decidir se um requisito específico é arquiteturalmente significante é normalmente uma questão de julgamento. Normalmente, eles são requisitos tecnicamente desafiantes, tecnicamente restritivos ou centrais para a finalidade do sistema.

Bons exemplos de Requisitos Arquiteturalmente Significantes são:

  • O sistema deve gravar cada modificação nos registros do cliente para finalidade de auditoria.
  • O sistema deve responder dentro de 5 segundos.
  • O sistema deve ser implantado no Microsoft Windows XP e no Linux.
  • O sistema deve encriptar todo o tráfego da rede.
  • O sistema ATM deve dispensar o dinheiro sob demanda para portadores de contas validadas e com saldo suficiente.
