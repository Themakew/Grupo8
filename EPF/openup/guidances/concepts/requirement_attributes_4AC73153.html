Concept: Atributos dos Requisitos

Main Description

Os Atributos dos Requisitos são as propriedades de um requisito. Os atributos capturam importantes informações adicionais de cada requisito. Estas informações podem ser utilizadas para responder a pesquisas sobre o status do projeto de desenvolvimento.

Exemplos de atributos

Segue abaixo uma lista de atributos normalmente usados em pequenos projetos, acompanhados de uma descrição resumida dos seus significados. Alguns atributos são mais bem descritos como números, enquanto outros são mais bem descritos como datas, valores booleanos (verdadeiro ou falso) ou campos de texto livre. Outros atributos podem ser expressos como listas enumeradas. Por exemplo, a prioridade pode ter valores como alta, média ou baixa.

Prioridade - Declaração da importância relativa do requisito para os Stakeholders (alta, média, baixa).

Atribuído a - Quem na organização é responsável por certificar que o requisito foi satisfeito (nome da pessoa ou da organização).

Iteração Alvo - A iteração na qual está planejado que o requisito será implementado (número ou texto).

Estimativa de tamanho - lhe dá uma estimativa de alto nível para o esforço necessário para implementar e verificar o requisito, normalmente medida usando uma unidade neutra tal como pontos. Veja Guideline: Estimativa Ágil para mais informações.

Esforço Restante - Uma estimativa do esforço necessário para implementar e verificar os requisitos restantes (horas).

Status de Conclusão - O progresso da implementação de um requisito. Pode ser capturado como uma lista enumerada (Completo, Parcialmente Concluído, Não Iniciado) ou pode ser inferida a partir do atributo Esforço Restante.

Quando estes atributos têm valores atribuídos para cada requisito, torna-se fácil responder a perguntas típicas sobre o projeto, tais como:

Quantos requisitos deverão ser implementadas na iteração atual?
 (consulte o atributo Iteração Alvo)

Qual a percentagem dos requisitos de alta prioridade que já está implementada?
 (consulte os atributos Status de Conclusão e Prioridade)

Quantos requisitos atribuídos à iteração atual ainda não foram implementados?
 (consulte os atributos Iteração Alvo e Status de Conclusão)

Quais requisitos estão atribuídos para mim?
 (consulte o atributo Atribuído a)

Exemplos de outros atributos de requisitos úteis são:

Origem - Pessoa, documento ou outra fonte de um determinado requisito. É útil para determinar a quem fazer perguntas ou agrupar requisitos de acordo com a pessoa que faz a solicitação.

Comentários - Comentários do revisor ou autor sobre o requisito.

Dificuldade - Uma indicação do nível de esforço necessário ou quão difícil será implementar o requisito (alta, média, baixa).

Risco - Medida de certeza sobre a satisfação (ou não) do requisito. Pode ser alto, médio, baixo ou números inteiros numa escala de zero a dez.

ID do Teste - Identificação de um teste específico ou outro método de verificação.
