Guideline: Submetendo Solicitações de Mudança
Esta diretriz descreve o tipo de informação que é normalmente capturada em uma solicitação de mudança. Esta informação é usada para priorizar e definir o escopo do trabalho necessário para implementar a mudança e monitorar o progresso.

Main Description

Visão Geral

As solicitações de mudança normalmente têm um ciclo de vida. São criadas, revistas, aceitas ou rejeitadas, implementadas, verificadas e finalizadas. Estes estados definem o status da solicitação de mudança em um ponto no tempo e representam uma informação crítica para o acompanhamento do progresso. Outros tipos de estados também são possíveis.

Durante a revisão de uma solicitação de mudança, o objetivo é avaliar o impacto da implementação da mudança tanto técnico como de custo e no cronograma. A avaliação técnica de impacto inclui a determinação de quais produtos de trabalho serão afetados e uma estimativa do nível de esforço necessário para alterar e verificar todos os artefatos afetados. Esta informação torna-se a base para as avaliações de custo e de impacto no cronograma e, finalmente, se a solicitação de mudança será aceita ou rejeitada.

Se for aceita, a implementação e a verificação da mudança solicitada serão atribuídas a uma iteração da mesma forma que qualquer outro item de trabalho.

O processo atual usa a Artifact: Lista de Itens de Trabalho para capturar, priorizar e acompanhar as solicitações de mudança usando os atributos definidos para esse artefato.

Submetendo Solicitações de Mudança

Ao submeter uma solicitação de mudança forneça o máximo de informação possível para permitir uma rápida revisão e disposição. No mínimo, todas as solicitações de mudança devem incluir as seguintes informações:

  • ID - um identificador único da solicitação de mudança para simplificar o seu rastreamento. Se você estiver usando algum tipo de ferramenta de acompanhamento de mudança a ferramenta deverá atribuir um ID único.
  • Descrição Resumida - Um nome que resuma a solicitação de mudança.
  • Descrição Detalhada - Uma descrição detalhada da solicitação de mudança. Para um defeito, se você puder fornecer informações que ajudem a reproduzi-lo, faça-o. Para uma solicitação de melhoria, forneça o raciocínio lógico para a solicitação.
  • Item Afetado - O artefato afetado e sua versão.
  • Severidade - Quão severa é esta questão (Super importante, é bom ter, etc.).
  • Prioridade - Quão importante é esta solicitação em sua opinião.

Dependendo do sistema que você estiver usando, os nomes destes elementos de dados podem diferir. Entretanto, esta informação é necessária, de uma forma ou de outra, para permitir uma rápida revisão e disposição da solicitação de mudança.
