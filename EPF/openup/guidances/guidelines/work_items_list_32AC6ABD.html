Main Description
O Artifact: Lista de Itens de Trabalho captura todo o trabalho que deve ser executado no projeto, bem como o trabalho proposto que pode afetar o produto. Alguns dos Itens de Trabalho podem ser implementados neste projeto, alguns deles podem ser implementados em um projeto futuro, e outros podem nunca ser implementados.

Alguns Itens de Trabalho podem ser mal definidos, portanto, eles podem representar uma grande quantidade de trabalho que pode requerer diversos meses de esforço da equipe de colaboradores. À medida que a prioridade destes Itens de Trabalho aumenta, eles são decompostos em itens de trabalho menores que representam tarefas específicas e bem definidas e que podem levar horas ou de dias para serem tratadas, veja Micro-Incrementos. Em outros casos, Itens de Trabalho específicos e bem definidos são criados diretamente, representando, por exemplo, um defeito a ser tratado, como mostra a figura.

A Lista de Itens de Trabalho fornece uma lista priorizada do trabalho proposto e agendado.



Um Item de Trabalho pode representar o trabalho associado a um defeito, uma solicitação de melhoria, uma solicitação de mudança, um caso de uso, um cenário, uma história do usuário, um requisito suplementar ou qualquer outra coisa que capture um requisito ou uma melhoria para o sistema. Um Item de Trabalho pode referenciar qualquer tipo de requisito ou outra informação útil que lhe direcione sobre o que deve ser feito.

Uma grande vantagem do Artifact: Lista de Itens de Trabalho é que ele permite priorizar uma única lista que contenha todas as coisas que necessitam ser tratadas, independente do Item de Trabalho representar um trabalho relacionado a um requisito, uma melhoria ou um defeito. A única exceção é que a lista de riscos seja priorizada separadamente.

Nada no projeto será feito se não estiver representado ou mapeado em um Item de Trabalho. Isto significa que todos os requisitos, relatos de defeitos e solicitações de mudança tem que, em algum momento, ser mapeados para um Item de Trabalho. Um desenvolvedor, também, não poderá trabalhar o que não estiver representado em um Item de Trabalho. Somente Itens de Trabalho precisam ser priorizados. Isto também significa que o rastreamento de Itens de Trabalho é a primeira forma de entender o status do projeto.

Existem dois principais tipos de Itens de Trabalho:

  • Itens de Trabalho Não Agendados: Estes Itens de Trabalho ainda não foram atribuídos a uma iteração, e não há nenhuma estimativa detalhada de esforço para eles.
  • Itens de Trabalho Agendados: Estes Itens de Trabalho estão atribuídos a uma iteração, e têm um conjunto adicional de atributos preenchidos tais como estimativas detalhadas de esforço.

Itens de Trabalho Não Agendados

A maioria dos Itens de Trabalho são inicialmente não agendados, significando que ainda não foi decidido se eles serão feitos, e quando serão. Os Itens de Trabalho não agendados devem sempre representar algo significativo a ser entregue aos Stakeholders, tal como um cenário a ser detalhado, projetado, implementado e testado. Você pode considerar a captura dos seguintes dados para tais Itens de Trabalho:

  • Nome
  • Descrição
  • Prioridade
  • Tamanho estimado, tal como uma estimativa de pontos (veja Guideline: Estimativa Ágil).
  • Novo, Atribuído, Resolvido, Verificado, Terminado (veja abaixo os Estados de Itens de Trabalho)
  • Links para outro material de referência, tal como requisitos ou especificações detalhadas do que é necessário ser feito

Itens de Trabalho Agendados

Após um Item de Trabalho ter sido atribuído a uma iteração, ele se torna agendado. Nós atribuímos os Itens de Trabalho somente à iteração atual ou seguinte. Não há nenhuma necessidade em atribuir Itens de Trabalho a uma iteração futura, visto que não é possível predizer que um cronograma terá mais do que uma iteração (veja Guideline: Planejamento da iteração).

Os atributos adicionais a seguir são úteis para Itens de Trabalho Agendados:

  • Iteração Alvo
  • Membro da equipe responsável
  • Estimativa de esforço, tal como horas reais de trabalho (veja Guideline: Estimativa Ágil).
  • Horas trabalhadas

Os atributos acima fornecem a informação necessária para planejar e controlar uma iteração. Nós podemos planejar iterações pelo conhecimento do esforço envolvido e também fazer um Report: Relatório de Progresso da Iteração acompanhando quanto trabalho ainda falta fazer.

Estados do Item de Trabalho

Nós achamos que os seguintes estados são úteis para acompanhar os Itens de Trabalho:

  • Novo: O Item de Trabalho foi criado, mas ainda não foi atribuído a um membro da equipe.
  • Atribuído: Um membro da equipe foi identificado como responsável pelo Item de Trabalho.
  • Resolvido: O membro da equipe responsável pelo Item de Trabalho implementou e testou o Item de Trabalho.
  • Verificado: O Item de Trabalho foi testado de forma independente.
  • Terminado: O Item de Trabalho não está mais ativo.

Você pode escolher um outro grupo de estados, baseado em suas necessidades.
