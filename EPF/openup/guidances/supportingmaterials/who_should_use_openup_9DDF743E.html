Quem pode usar o OpenUP?
Este material de apoio explica quem pode usar o OpenUP?
Main Description

O OpenUP é mais útil para quatro grupos primários de usuários:

  • Profissionais de desenvolvimento de software (desenvolvedores, gerentes de projeto, analistas e testadores) que trabalham juntos como uma equipe de projeto;
  • Stakeholders;
  • Engenheiros de processo de software;
  • Instrutores.

Os profissionais de desenvolvimento de software podem encontrar orientação sobre o que será exigido deles quando da execução dos papéis definidos no OpenUP. Cada papel descreve um conjunto de atividades e artefatos de sua responsabilidade. Também é dada orientação de como estes papéis colaboram entre si.

Os Stakeholders encontrarão orientação do que podem esperar da equipe de desenvolvimento de software e como o software será criado. O OpenUP descreve também as responsabilidades dos Stakeholders e qual a melhor forma deles trabalharem com a equipe de desenvolvimento para obter um software que atenda às suas necessidades.

Os engenheiros de processo de software podem usar o EPF Composer para modificar e estender o OpenUP. As modificações podem ser tão simples como as alterações de templates para produtos de trabalho ou tão sofisticadas como a adição de atividades necessárias para criar o software em seu ambiente específico (tais como auditorias para sistemas de missão crítica). Além da modificação do conteúdo de método, os engenheiros de processo podem incluir, alterar ou excluir fluxos de processo para adicionar padrões de capacidade específicos da organização.

O OpenUP também se aplica às organizações acadêmicas. Como um processo de código aberto, pode servir como base para cursos de engenharia de software e, quando combinado com o EPF Composer, para cursos de engenharia de processo de software.
