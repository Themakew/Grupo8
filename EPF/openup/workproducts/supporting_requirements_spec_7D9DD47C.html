Description
Main Description

Juntos, os Requisitos Suplementares e os Casos de Uso definem todos os requisitos do sistema. Os Casos de Uso descrevem os requisitos comportamentais do sistema, e os Requisitos Suplementares descrevem todos os outros requisitos que não são capturados nas Especificações de Caso de Uso. Ao fazer esta distinção simplificamos a manutenção.

Os Requisitos Suplementares podem ser categorizados de acordo com o modelo FURPS+ (Funcional, Usabilidade, Confiabilidade, Desempenho, Suportabilidade + Restrições). Para mais informações sobre esta classificação, veja Concept: Requisitos Suplementares.

Tailoring
Impact of not having

O objetivo deste produto de trabalho é certificar que todos os tipos de requisito estão cobertos, o que reduz o risco da não consideração de alguma característica importante do sistema. Os requisitos FURPS+ são gerais ao sistema, e influenciam os Mecanismos Arquiteturais que você irá criar, guiando assim o desenvolvimento das fundações do sistema. Estes requisitos são normalmente os principais itens de custo, porque eles determinam as escolhas arquiteturais.

Além disso, se você não capturar os requisitos gerais ao sistema em um lugar central, mas repeti-los em todos os Casos de Uso, o resultado implicará em mais manutenção e mais possibilidades de erro.
Representation Options

Recomendação: Use o Template de Especificação de Requisitos Suplementares

O template de Especificação de Requisitos Suplementares fornece uma ferramenta para registrar, estruturar e organizar os Requisitos Suplementares.

Mesmo em um projeto pequeno, uma ferramenta de gestão de requisitos, uma base de dados ou uma planilha, são recomendadas para priorizar e controlar os requisitos. Se os Stakeholders estiverem confortáveis em acessar os requisitos diretamente a partir dessa ferramenta ou acessar um informe gerado automaticamente pela ferramenta, você não necessitará de um documento em separado.

Opção: Usar a Lista de Itens de Trabalho

Considere o registro dos Requisitos Suplementares na Lista de Itens de Trabalho, que você pode usar para priorizar e controlar os requisitos. Se os Stakeholders estiverem confortáveis em acessar os requisitos diretamente a partir da Lista de Itens de Trabalho ou acessar um informe gerado automaticamente a partir dela, você não necessitará de um documento em separado.

Opção: Incluir como Parte do Documento de Visão

Considere incluir alguns tipos de Requisitos Suplementares no documento de Visão. Para manter a Visão estável, use esta opção para os tipos de requisitos que necessitem de menos refinamento, tais como Qualidades do Produto, Documentação ou Conformidade.
